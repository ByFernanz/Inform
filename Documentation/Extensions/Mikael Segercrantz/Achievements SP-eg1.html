Extensions

 Achievements SP by Mikael Segercrantz

Version 1.0.60808

A table-based way to assign scores for actions, rooms and objects. Spanish adaptation by Mel Hython.

*** Traducción de la documentación: la documentación en inglés original se encuentra tras esta traducción ***

La extensión Achievements permite un sistema de puntuación simplificada para los juegos. Permite diferentes formas de puntuar, como por ejemplo encontrar ciertos objetos y localizaciones. Todos lo objetos y localizaciones obtenidas que sirvan para puntuar son manejados mediante tablas: la tabla de Table of Achievements, la tabla de Table of Scored Objects, y la tabla de Table of Scored Places.

La extensión define dos variables globales:

1. un variable de tipo número que se llama Achievement-Init. La extensión la usa para saber si ha sido inicializada o no, ya que requiere inicializarse a sí misma cuando se produce la descripción de la primera localidad del juego (no se puede usar 'when play begins' ya que no funciona debido a que la localización del jugador en ese momento es la dark room y no la localización 'real'). La inicialización garantiza que los puntos de la habitación inicial, si hay alguna, así como los objetos ya obtenidos en el principio de la historia son puntuados si debe ser así. El nombre se ha escogido para intentar minimizar el riesgo de colisión con otras incluídas en el código por el escritor de la historia.
2. una variable de tipo número que se llama achievement maximum score que contiene la puntuación máxima obtenible y que se calcula de forma automática en tiempo de inicialización.

Para usar la extensión necesitas 'continuar' las tablas que quieras usar.

La tabla de puntuar objetos se definiría como sigue:

Table of Scored Objects
object     points     used
a thing     a number     a number

Esta tabla contiene los objetos que proporcionan puntos. La columna 'object' indica el nombre del objeto que puntua, la columna 'points' los puntos obtenidos por él y la columna 'used' (que será inicializada a 0 en el comienzo del juego) simplemente permite saber si se ha puntuado o no por ese objeto en particular.

La tabla de puntuar lugares se define como:

Table of Scored Places
loci     points     used
a room     a number     a number

Que es similar a la tabla de puntuar objetos. La columna 'loci' debe contener la habitación que puntua. 'points' contiene los puntos a puntuar. Y 'used' es una marca como en el caso anterior.

La tabla de objetivos generales se define como:

Table of Achievements
message     points     used
text     a number     a number

Que es más o menos lo mismo que las anteriores. El texto se muestra durante la lista de puntuaciones.

Para puntuar un objetivo general debe usarse la sintaxis:

score the achievement with message "the message entry";

lo que hará que se puntue esa entrada.

Es posible verificar si un objetivo general, un objeto o una localización han sido puntuadas usando:

the achievement "<message>" is scored
the item <object> is scored
the room <room> is scored

en donde <message> es el texto de la tabla, <objeto> es un objeto de la tabla, y <room> una habitación de la tabla.

En caso de ser necesario se puede redefinir la regla de cada turno, "scoring of rooms and objects".

Esta extensión reemplaza el manejo de puntuación de I7, incluyendo la forma de describir la puntuación final.

*** Original english documentation ***

The Achievements extension allows for ease of scoring in a game. It allows for different achievements to be scored, as well as finding certain items and rooms. All the achievements, items, and rooms for scoring are managed through tables; the Table of Achievements, the Table of Scored Objects, and the Table of Scored Places respectively.

The extension defines two global variables:

1. a number which varies called Achievement-Init. This is used for the extension to know if it has been initialized or not, since it needs to initialize itself when beginning the description of the first location in the game (when play begins does not work, unfortunately, since the location of the player during it is the dark room, not the real location). Initialization grants the points from the start room, if it has any, as well as of any objects enclosed by the player at the beginning of the game, if any have scores defined on them. The name has been chosen to minimize the risk of clashing with a variable in user code.
2. a number which varies called the achievement maximum score, which is automatically calculated at initialization time.

To use the extension, you need to continue the tables you want to use.

The Table of Scored Objects is defined as follows:

Table of Scored Objects
object     points     used
a thing     a number     a number

The Table of Scored Objects contain the objects for which a score will be given. The object column is the name of the object, the points column the value of the object, and the used column (which will be initialized to 0 at the start of the game) marks the fact of having received the points from the object.

The Table of Scored Places is defined as follows:

Table of Scored Places
loci     points     used
a room     a number     a number

As with the Table of Scored Objects, the Table of Scored Places has the same function. The loci column defining the room to be scored replaces the object column. The used column will be initialized to 0 automatically.

The Table of Achievements is defined as follows:

Table of Achievements
message     points     used
text     a number     a number

Again, the table looks like the previous tables, and again, the used column will be initialized to 0 automatically. The message contains the text which will be shown while listing the full score.

To score an achievement, you need to use the following syntax:

score the achievement with message "the message entry";

and the points will be added to the score.

It is possible to test if an achievement, an object or a room has been scored, using the phrases

the achievement "<message>" is scored
the item <object> is scored
the room <room> is scored

where <message> is the message entry used to score the achievement, <object> is the item tested for and <room> is the room tested for.

If needed, you may override the every turn rule, "scoring of rooms and objects".

The extension replaces the score and the full score handling of Inform 7, including the final score displaying, to manage the listing of scores from the tables, and contains a little bit of code in Inform 6 - code neutral to the Z-machine and Glulx.

 Example Puntuando

Un ejemplo variado de puntuaciones.

"Puntuando"

Include Achievements SP by Mikael Segercrantz.

Creamos unas habitaciones.

The entrada is a room.
The basemento is a room.
The comedor is a room.

Puertas y conexiones:

A puerta de madera is a door. It is west of the entrada and east of the comedor. It is closed and openable.

The basemento is below the entrada.

Y un objeto que puntue:

A gorra de beisbol is a thing. Some cajas are in the basamento. They are fixed in place.

Instead of searching the cajas for the first time:
     now the gorra de beisbol is in the basamento;
     say "¡Rebuscando entre las cajas encuentras una gorra!"

After opening the puerta de madera, score the achievement with message "encontrar la salida".

And the tables:

Table of Achievements (continued)
used     points     message
0     10     "encontrarla salida"

Table of Scored Places (continued)
used     points     loci
0     5     basamento
0     5     comedorl

Table of Scored Objects (continued)
used     points     object
0     5     gorra de beisbol

Cambiar la barra de estado:

When play begins:
     change the right hand status line to "[score] de entre [achievement maximum score]".

Final del juego:

Every turn:
     if the score is the achievement maximum score, end the game in victory.

Testeo:

Test me with " abrir puerta / oeste / este / abajo / buscar en las cajas / coger gorra ".
